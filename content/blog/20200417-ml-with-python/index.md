機械学習の学習に着手した。  
立ち読みレベルで巷にあふれる Python + 機械学習なテーマの本を立ち読みしたものの内容が浅く、コードも実用性に乏しいのが多かった。  
結局、下手に市販の参考書を買うことはせず、いきなり Kaggle の Notebook を漁るスタイルに落ち着く。  
雑に、 Kaggle の Notebook で votes 順 & Language == Python でソートして上から読んだだけだが。  
データサイエンティストの作業が追体験でき、超速で機械学習のノリ(背景の理論ではなく、表層のコーディング)がつかめるのでとても良かった。  
コードも良さげなものについてはスニペットとして吸収しておく。  

いろいろと厳密ではないが、個人的には機械学習 == "多次元の特徴量を含んだ入力を、それが属するクラス(あるいは値)に射影する関数 f を求めること" と解釈。  
データの分布、特性によって関数の選び方にノウハウが有るので、特徴量エンジニアリングだったりモデルの選定だったり、の知識が必要になってくる。  

流行りの深層学習は特徴量エンジニアリング不要！みたいなノリだが、重厚なニューラルネットで学習に時間がかかる。  
hyper parameter の最適化をしようことなら計算いつ終わるのレベル。  
今の所実務には使いづらそうだなーという印象だが、TensorFlow とその周辺のツール群(TensorFlow Serving, TensorBoard 等々)が充実しているので、そのうち言ってることが変わるかもしれない。  

深層学習の文脈で言う、ニューラルネットワークの学習は誤差逆伝播法が背景に有り、出力から入力へ微分値を計算して重みの変更する方向(コスト関数が減少する方向)を求める。  
なぜこのような計算をするかの直感的な説明は、[ゼロから作るDeep Learning ―Pythonで学ぶディープラーニングの理論と実装](https://amzn.to/34KhGXB)が詳しい。合成関数の微分を絡めつつ説明するとなんとなく腑に落ちる。あくまでなんとなく。

実務への適用を考えたとき、実は Deep Learning よりも Kaggle でよく使われている手法(XGBoost, Lightgbm など)は抑えておくべき。  
決定木、ブースティング、アンサンブル手法なんかは、地味に実用性に富んでいて、結果も解釈しやすいことから今後お世話になりそう。

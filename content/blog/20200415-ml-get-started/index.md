# Docker コンテナ上に Jupyter Notebook サーバを構築

* クライアント PC にライブラリ群をインストールする必要がない
* 環境構築が容易

というメリットから、Docker コンテナ上に Jupyter Notebook サーバを構築することにした。  
これにより、ブラウザから Python スクリプトをインタラクティブに実行できるようになった。  

# コードベース

Jupyter Notebook は作業ログとしては有用だが、自動化には向いていない。  
諸々自動化したいのでスクリプトに落とし込む。  
データI/O・前処理・モデル生成・学習処理・評価等、各フェーズをモジュール分割しておき、  
メインスクリプトからはその関数群を呼び出す構成にしたい。  
この際、各モジュールは差し替え可能にしておくことで、  
例えば csv から DB サーバに変わったとなればデータ I/O モジュールを書き換え、  
学習モデルを変更する場合はモデル生成のみを差し替え、  
といった感じに柔軟に作業ができるようになるはず。  
設計は随時改善されそうなので、案件ごとにディレクトリを掘って別管理するものとし、無理な抽象化は避ける。
